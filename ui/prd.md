## 1. 产品概述

### 1.1 项目背景
基于 Flutter 框架构建一套可在 Android、Windows、Linux、macOS 上运行的 core 代理客户端。旨在提供统一核心逻辑、高颜值界面，并针对移动端和桌面端的操作习惯进行差异化但逻辑一致的交互设计。

### 1.2 产品目标
- **核心统一**：复用 core 核心调用逻辑与状态管理代码，仅 UI 层针对平台进行适配。
- **移动优先 (Android)**：强调单手操作便捷性，核心操作（连接/断开）位于右下角拇指热区。
- **桌面沉浸 (Desktop)**：保持界面简洁，通过系统托盘常驻后台，不占用任务栏过多空间。

### 1.3 核心技术前提
- **Tun 模式**：core 作为独立进程运行，创建 Tun 设备接管系统流量。
- **无内置 Geo 数据**：core 编译时未包含 GeoIP.dat / GeoSite.dat，必须通过指定远程 URL 下载 **Domain 列表** 和 **CIDR 列表** 文件来实现路由规则。

## 2. 用户角色
- **技术爱好者**：需要切换不同的入站/出站协议，查看详细日志。
- **日常翻墙用户**：关注一键连接速度和订阅更新。

## 3. 功能需求详述


### 3.1 通用核心逻辑
- **core 核心管理**：后台运行 core 进程（桌面端）或 Service/VPNService（Android 端）。
- **状态同步**：连接状态（已连接、断开中、已断开、延迟数值）在多端 UI 上实时反馈。
- **配置解析**：支持 VMess、VLESS、Trojan、Shadowsocks 等主流协议，支持 `v2rayN` 格式订阅链接解析。

### 3.2 Android 端详细设计

#### 3.2.1 主页布局
遵循 Material Design 3 规范。

| 区域 | 组件 | 功能描述 |
| :--- | :--- | :--- |

#### 3.2.3 子界面设计
- **订阅设置界面**：列表展示订阅源，显示最后更新时间、流量使用情况（若有），支持手动更新按钮。
- **应用设置界面**：单选框/开关控件，配置预定义规则（如“绕过中国大陆 IP”）。
- **路由设置界面**：高级配置编辑器，或提供预设规则选项（如：广告屏蔽、全局代理、仅代理国外）。

#### 3.2.4 分应用代理详细交互
- 进入 **应用设置** -> **分应用代理**。
- 展示系统已安装应用列表（含应用图标、包名），支持搜索。
- 提供两种模式切换：
    - **白名单模式**：仅代理列表中勾选的应用。
    - **黑名单模式**：代理除勾选外的所有应用。
- 保存后，Flutter 将包名列表传递给原生层，在启动 VPN 时通过 `VpnService.Builder` 配置。


### 3.3 桌面端详细设计 (Windows / Linux / macOS)

#### 3.3.1 主窗口策略
- **启动行为**：应用启动后默认**隐藏主窗口**，仅在系统托盘显示图标。
- **窗口模式**：提供简化的主窗口视图（用于节点管理和编辑），支持点击关闭按钮时**最小化至托盘**而非退出。

#### 3.3.2 系统托盘交互 (Tray Menu)
这是桌面端的核心操作入口，设计需等同于 Android 端的“主页+抽屉”。

| 托盘菜单区域 | 菜单项 | 功能映射 |
| :--- | :--- | :--- |
| **状态栏** | (不可点击) | 显示：`[节点名] (延迟: 100ms)` |
| **连接控制** | **连接 / 断开** | **等同于 Android 右下角 FAB**。点击后状态同步变更。 |
| **快速切换** | 最近使用节点列表 | 展示最近的 3-5 个节点，点击直接切换并自动连接（若处于连接状态则重连）。 |
| **系统设置组** | **订阅管理**<br>**路由配置**<br>**日志查看**<br>**偏好设置** | 点击各项会**唤起主窗口**并导航至对应的配置页面。<br>*(遵循桌面端习惯，配置类操作在主窗口完成)* |
| **底部操作** | **显示主窗口 / 隐藏**<br>**退出程序** | 控制窗口可见性与彻底退出。 |

#### 3.3.3 主窗口 UI 布局 (桌面端特有)
当用户从托盘点击“订阅管理”或双击托盘图标时，显示此窗口：

- **左侧导航栏 (Rail)**：显示图标（服务器列表、订阅管理、路由设置、日志）。
- **右侧内容区**：展示对应的配置界面，设计风格与 Android 子界面保持一致，但使用更大的鼠标点击热区。

#### 3.3.4 主窗口路由设置界面
- 提供 **“规则文件源”** 管理卡片：显示当前使用的 Domain 列表文件路径和 CIDR 列表文件路径。
- 支持输入 URL（如 `https://example.com/domain-list.txt`）并点击下载按钮，自动保存至本地应用数据目录。
- 勾选框：**启用自定义路由规则**（对应 core 配置中 `routing.domainStrategy` 及规则项）。


## 4. 技术实现与架构建议 (Flutter)

### 4.1 平台通道 (Platform Channel) 职责划分

| 功能点 | Flutter 层职责 | 原生层职责 (Android / Desktop) |
| :--- | :--- | :--- |
| **core 核心** | 发送配置 JSON，接收状态流 | 启动/停止 `core` 可执行文件或库，建立 VPN 隧道。 |
| **系统托盘** | 构建菜单数据结构 (Dart) | **Desktop**: 调用 `tray_manager` 插件渲染原生菜单。<br>**Android**: 无此需求。 |
| **VPN 权限** | 触发流程 | **Android**: 弹出 `VpnService.prepare()` 授权页。 |
| **网络流量统计** | 定时轮询接口 | 读取本地 core API 端口 (如 8081) 的 `stats` 数据。 |

- 
### 4.2 推荐 Flutter 插件选型
- **状态管理**: Riverpod (适合多窗口与后台状态同步)。
- **桌面托盘**: `tray_manager` (支持 Linux, Windows, macOS)。
- **本地存储**: `shared_preferences` (简单配置) + `sqflite` (节点列表与订阅缓存)。
- **UI 组件**: `flutter_slidable` (列表滑动操作), `window_manager` (控制窗口隐藏/显示/关闭行为)。

### 4.3 Tun 设备与分应用代理实现（Android 原生层）

**原理说明**：
- core 可执行文件启动时会创建一个 Tun 虚拟网卡（如 `tun0`）。
- Android 端通过继承 `VpnService` 建立 VPN 连接，将设备流量路由至该 Tun 接口，再由 core 进程处理。
- **分应用代理**利用 `VpnService.Builder` API：
    ```java
    // 白名单模式示例
    builder.addAllowedApplication("com.example.app1");
    builder.addAllowedApplication("com.example.app2");
    
    // 黑名单模式示例
    builder.addDisallowedApplication("com.android.browser");
    ```
- Flutter 通过 MethodChannel 调用原生方法，传入包名列表及模式标志，原生层在 `onStartCommand` 中构建 VPN 接口时应用该配置。


### 4.4 API Inbound 接口实现说明

为了实时获取流量统计、切换节点配置等，Core 需开启一个 `api` 入站协议。
```yaml
inbounds
- tag: api
  port: 10085
  listen 127.0.0.1
  protocol: api
```

#### 4.4.1 接口功能说明（Flutter 调用）

| 功能 | HTTP 端点 | 请求体示例 | 说明 |
| :--- | :--- | :--- | :--- |
| **查询流量统计** | `GET /stats` | - | 返回所有计数器，Flutter 解析上行/下行总流量。 |
| **查询指定代理出站流量** | `GET /stats?name=outbound` | - | 获取具体 outbound 标签的流量。 |
| **切换出站节点** | `POST /handler` | `{ "tag": "outbound-1" }` | 调用 HandlerService 切换当前默认出站。 |

- **Flutter 实现**：使用 Dart `http` 库请求 `http://127.0.0.1:10085/...`，定时轮询（例如每秒）更新界面速率显示。
- **安全注意**：API 端口仅监听本地回环地址。

### 4.5 路由规则加载（无 GeoIP/GeoSite 方案）

由于 core 核心不支持内嵌 geo 数据，路由配置需引用外部规则文件，并通过远程 URL 下载更新。

#### 4.5.1 规则文件格式要求
- **Domain 列表文件**：纯文本，每行一个域名，支持 `domain:example.com` 或 `full:www.example.com` 格式。
- **CIDR 列表文件**：纯文本，每行一个 IP 段，如 `1.1.1.0/24`。

#### 4.5.2 Flutter 职责
- 在 **路由设置界面** 提供 URL 输入框，用户点击 **“更新规则”** 后，Flutter 使用 `Dio` 下载文件并保存至 `getApplicationDocumentsDirectory()`。
- 启动 core 进程前，将本地文件路径动态写入生成的配置 JSON 中。

### 4.6 桌面端系统托盘与进程管理

- 使用 `tray_manager` 插件创建原生托盘图标及菜单。
- 使用 `process_run` 或 `dart:io` 启动/停止 core 子进程，通过 PID 管理。
- 主窗口关闭时隐藏而非退出，确保后台进程继续运行。

## 5. 非功能性需求

### 5.1 性能
- 内存占用：桌面端空闲状态 < 50MB，Android 后台运行 < 80MB。
- 启动速度：Flutter 引擎加载完成后 1 秒内完成节点列表渲染。

### 5.2 适配性
- **Android**：最低支持 API Level 24 (Android 7.0)。
- **macOS**：支持 Intel 和 Apple Silicon 双架构打包。
- **Linux**：支持 AppImage 格式分发。

### 5.3 安全性
- 配置文件必须进行 AES 本地加密存储，防止明文泄露节点密钥。
- 桌面端退出时自动断开连接，防止代理残留。


### 5.4 兼容性
- **Android**：API 24+，需声明 `android.permission.BIND_VPN_SERVICE`。
- **桌面**：需管理员/root 权限创建 Tun 设备（Linux 需配置 `setcap`）。


## 6. 改进点总结 (相较于原始需求)

1.  **Android 搜索优化**：明确右上角 `+` 包含扫码/粘贴功能，搜索独立出来用于过滤长列表。
2.  **桌面端防误操作**：主窗口关闭默认隐藏至托盘，明确退出必须在托盘菜单执行，符合 Windows/macOS 后台程序惯例。
3.  **状态感知**：桌面端托盘菜单**顶部**显示当前连接状态和节点名，避免用户盲目点击连接按钮。
